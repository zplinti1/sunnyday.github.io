了解的越多，越觉得自己无知

2018-11-17

花了11个小时，把极客时间出品的《Java核心技术36讲》过了一遍，有两个大的感受：

  • Java的世界，真是浩瀚的大海，我写过几行代码，见过几个小池塘，几条小溪，便想象Java的世界，最多也就是几个小湖泊，真是见识短浅。
  • 小溪最终也会汇入湖泊海洋，我如果对用到的东西，多一分好奇，或许早就见到江河湖海，日月星辰。

Java Exception的小故事

记得我刚加入LogicMonitor不久，那时，我还在从C切换成Java。我的boss在review我的代码时，发现我对Exception和Error的概念完全不清楚，给我简单讲解几句后，然后给我几个文档去学习，那时起，我才对它们有一些初步了解。

  • jenkov - Java Exception Handling
  • ImportNew - 深入理解java异常处理机制

想来那时，啥也不懂，也不会觉得自己无知，每天写着各种业务代码，倒也是轻松愉快，就这样过了几年，回头一看，发现其实自己并没有太多成长。36讲里，我能答得上来的，不超过5讲。

这是一个坏事，我度过这么多年光阴，没有太多成长。
这是一个好事，我花了11个小时，就对这36讲里的内容有初步的了解。

JMH的小故事

有缘的是，前日中午，组内有个同事做了个技术分享，给我们分享JMH， 而第二天看到第34讲，也恰好介绍这个，正好加深印象。

对比技术分享里的内容和课程里的内容，显然同事讲解的更加深入，更容易让我印象深刻。我不禁反思：

  1. 36讲，就这样看过一遍，查过一些附加资料，就算学习完了？更好的做法是，这只是一个起点，让我看到了星辰大海，而我如果对某个星星比较感兴趣，往后就可以继续深挖下去，再接着写个博客详细介绍，或者也做个技术分享。
  2. 为何同事比我更早的接触到JMH？回到开头，或许就是因为我对用到的东西，本着能用就行，需要深入了解时，再去花时间去看。工作的大部分情况，却是不需要我深入了解，或者说，没有人推动我去了解，我就不会去主动了解。前几年去看《Elasticsearch - The Definitive Guide》，也是因为boss推荐我去看，而后才去看；而Elasticsearch的源码，下载后就再也没有看了。做事的积极性、主动性，完全不够。当然现在好多了，用了iHour，用时间，记录自己的成长。虽然称不上“每天让自己博学一点点”，长期来看，随着时间流逝，我也越来越博学。

未来

36讲里的大部分内容，都是一些基础东西。计划把每一讲的内容，以及自己的领悟，用自己的话，也一一记录在这个博客里。这是个几倍于11个小时的工作量，哈哈，我又乱立Flag。
